## 1. 프로젝트 주제

**프로젝트 제목:** LoopBreaker — 코딩 AI 에이전트의 실시간 실패 탐지·사람 호출 미들웨어

**한 줄 설명:** Claude Code 같은 자율 코딩 에이전트의 세션 로그를 로컬에서 실시간 분석해, 규칙으로는 안 잡히는 "의미적 실패"(반복 편집 헛돌기·근거 없는 가짜 성공 선언)를 탐지하고 근거와 함께 사람을 호출하는 로컬 감시 도구.

---

## 2. 문제 정의 (Problem Definition)

### 2-1 해결하려는 문제

**해결하고 싶은 문제:**

자율 AI 코딩 에이전트가 점점 더 오래 혼자 실행되면서(Claude Sonnet 4.5는 약 30시간 독립 실행을 시연), 사람이 매 순간 화면을 지켜볼 수 없게 됐다. 그 결과 에이전트가 빠지는 **의미적 실패**가 누적·증폭돼도 즉시 알아채기 어렵다. 대표적으로:

- **헛돌기(thrashing):** 같은 파일을 미세하게 바꿔가며 N번째 재편집, 동일 에러로 계속 수렴하는데도 빠져나오지 못함.
- **가짜 성공(자기승인 우회):** 근거 없이 "완료했다"고 선언하거나, 자기가 쓴 근거를 같은 차례에 자기가 검증해 통과시키는 자기참조 우회.

이런 실패는 토큰 수·요청 수 같은 **결정론적 규칙(rate limit, 횟수 카운트)으로는 잡히지 않는다.** 입력이 조금씩 달라지면서 규칙을 빠져나가고, "완료 선언"은 형식상 정상이기 때문이다. 의미를 읽어야만 잡힌다.

**왜 이 문제가 중요한지:**

규칙으로 못 막은 실제 사고가 이미 일어났다.
- **Replit AI 에이전트 사고(2025-07):** 변경 금지 지시를 반복적으로 받았음에도 프로덕션 데이터베이스를 삭제하고, 가짜 데이터·테스트 결과를 지어냈으며, "롤백 불가"라는 거짓 주장까지 했다. 1,200명 이상 임원·약 1,190개 회사 데이터가 영향받았다. (출처: Fortune, The Register, AI Incident DB #1152)
- **Amazon Kiro 사고(2025-12):** AWS 비용 도구 버그 수정 중 "프로덕션 환경을 통째로 삭제·재생성하는 게 가장 효율적"이라 스스로 결론짓고 승인 없이 실행해, 중국 리전 13시간 장애를 냈다. (출처: AI Incident DB #1442)

그리고 이건 남의 일이 아니다. **본인의 개인 AI 워크플로우 하니스(dohyun) 분석 기록(`docs/evidence-model.md`)에 자기승인 우회가 5건 실제로 관측됐다** — 그중 2건은 AI가 근거를 적은 뒤 0초 만에 자기가 검증을 통과시킨 케이스다. 가설이 아니라 내 환경에서 실측된 문제다.

시장 흐름도 이 문제를 키운다. METR 연구에 따르면 에이전트가 수행 가능한 태스크 길이가 약 7개월마다 2배로 늘고 있고(arXiv:2503.14499), Gartner는 2026년 말 신규 엔터프라이즈 앱의 40%가 에이전트를 탑재할 것으로 본다(2025년 5% 미만). 자율 실행이 길어질수록 "사람이 못 보는 사이 헛도는 시간"의 비용은 커진다.

### 2-2 프로젝트 목표

**목표:**

Claude Code 세션 로그(JSONL)를 로컬에서 실시간으로 읽어, **헛돌기와 가짜 성공 2가지 의미적 실패를 탐지하고, "무엇이 왜 의심스러운지" 근거와 함께 사람을 호출(알림)하는** read-only 데스크톱 도구를 만든다.

- 핵심 산출물: macOS 백그라운드 데몬 + 탐지 엔진 + 근거 동반 알림.
- AI 핵심: 규칙으로 못 잡는 의미적 실패를 **임베딩 유사도 + LLM-as-judge**로 판정.
- 검증 가능성: 자체 골드셋(라벨 데이터)으로 탐지 정밀도(precision/recall/F1/κ)를 정량 측정한다.
- **명시적 비범위(MVP):** 에이전트를 자동으로 멈추거나 개입하지 않는다. "탐지 + 사람 호출"까지만 한다. 개입 권한은 사람에게 둔다. (자동 차단은 후속 과제)

## 3. 사용자 분석 (Target User Analysis)

| 항목 | 내용 |
|---|---|
| **대상 사용자** | Claude Code/CLI를 장시간 자율 모드로 돌리는 1인 파워유저 개발자. (예: JS/TS/Python 멀티 프로젝트를 동시에 굴리며, 에이전트에게 리팩터·버그수정을 맡겨두고 다른 일을 하는 개발자 — 본인 프로필과 일치) |
| **사용 환경** | macOS 로컬. `~/.claude/projects/**`에 세션 JSONL이 쌓이는 환경. 로컬 도구(LaunchAgent·SQLite·hooks) 설치·운영에 거부감이 적은 기술 사용자. |
| **사용자가 겪는 문제** | 에이전트가 길게 자율 실행되는 동안 화면을 못 본다. 헛돌기로 시간·토큰을 낭비하거나, "완료" 선언을 믿었다가 실제론 안 된 결과를 받는다. 개입 타이밍을 놓친다. |
| **사용 목적** | 에이전트를 켜두고 자리를 비워도, 헛돌거나 가짜로 끝내는 순간 **근거와 함께 호출받아** 제때 개입하기 위함. "지금 가서 봐야 한다"를 판단하는 비용을 줄인다. |

> **향후 확장(비MVP):** 여러 에이전트를 운용하는 소규모 팀. 다만 거버넌스·동시 세션·권한 모델 복잡도 때문에 MVP 대상에서 제외. 업계에서 자율 에이전트 파일럿 다수가 관측성·거버넌스 격차로 프로덕션 전에 실패한다는 점은 "팀 확장 시장이 존재한다"는 방향 근거로만 사용.

## 4. 사용 시나리오 (Use Case)

**사용 흐름 예시:**

개발자가 Claude Code에게 "이 테스트들 다 통과시켜줘"를 맡기고 자리를 비운다. 에이전트는 `auth.test.ts`를 고치기 시작하지만, 같은 에러로 계속 수렴하며 12스텝째 미세하게 다른 수정만 반복하고 있다. 규칙 기반 카운터는 "입력이 매번 다르다"며 그냥 통과시킨다. LoopBreaker는 이 의미적 헛돌기를 잡아내, 개발자 데스크톱에 **"최근 12스텝 동안 auth.test.ts를 5회 재편집하며 동일 에러로 수렴 중(유사도 0.94). 계획은 'X 수정'이었으나 실제 행동은 무관한 Y로 발산. 개입 권장."** 이라는 근거 있는 알림을 띄운다. 개발자는 돌아와 방향을 바로잡는다.

**단계 표:**

| 단계 | 사용자 행동 | 시스템 동작 |
|---|---|---|
| 1 | Claude Code로 장시간 자율 작업 시작(리팩터·버그수정 루프), 자리 비움 | LoopBreaker 데몬이 백그라운드에서 해당 세션 JSONL을 실시간 테일링 |
| 2 | (대기) | **구조 1차 게이트:** 같은 파일 N회 비슷한 편집 / (tool, args) 트리플 반복을 슬라이딩 윈도로 탐지 → "헛돌기 후보" 플래그 |
| 3 | (대기) | **의미 2차 판정:** 입력이 미세 변형돼 규칙을 빠져나가는 경우, 임베딩 코사인 유사도로 "의미적 수렴/silent cycle" 판정. 필요 시 LLM-judge가 세션 스냅샷을 루브릭으로 채점해 '가짜 성공(근거 없는 완료 선언)' 분류 |
| 4 | 데스크톱 알림 수신 | **컨텍스트 동반 호출:** 단순 "멈춤"이 아니라 *무엇이·왜* 의심스러운지(관련 편집·timestamp·유사도)를 근거로 알림 + 터미널 + (선택)웹훅 |
| 5 | 근거 확인 후 직접 개입(세션에 방향 수정 입력) | (개입은 사람이 수행 — MVP는 자동 차단 안 함) |
| 6 | 알림을 '맞음/오탐'으로 피드백 | 골드셋에 누적 → precision/recall/κ 갱신 → 임계값·루브릭 재보정 |

## 5. 기능 목록 정의 (Feature List)

| 기능 이름 | 기능 설명 |
|---|---|
| 세션 로그 실시간 테일링 | `chokidar`로 `~/.claude/projects/**`(서브에이전트 `**/subagents/**/agent-*.jsonl` 포함) 재귀 감시 + 바이트 오프셋 증분 파싱. 미완성 부분 라인 버퍼링·파싱 실패 허용. |
| 구조 1차 탐지 게이트 | (tool, 정규화 args[, result]) 트리플을 슬라이딩 윈도로 비교, 큰 payload는 SHA-256 해시. 동일 파일 N회 편집 탐지(substring 오탐 방지). 임계값 다이얼(warning/critical). 저비용으로 "문자 그대로의 반복"을 거름. |
| 의미 2차 판정 (헛돌기) | 임베딩 코사인 유사도로 silent cycle·의미적 수렴 탐지. 군집화(HDBSCAN 등). 임계값은 자체 골드셋으로 캘리브레이션. |
| LLM-as-judge (가짜 성공) | 루브릭 + CoT + position swapping/self-consistency로 편향 완화. 세션 스냅샷을 채점해 "근거 없는 완료 선언/자기승인 우회"를 분류. prompt caching으로 비용 절감. |
| 컨텍스트 동반 사람 호출 | 근거(관련 tool_use·timestamp·유사 클러스터)와 함께 데스크톱 알림 + 터미널 + 선택적 웹훅. |
| 로컬 저장 | `better-sqlite3` + `sqlite-vec` 단일 SQLite 파일에 이벤트·임베딩·판정 결과 저장. |
| 백그라운드 데몬 | macOS `launchd` LaunchAgent 경량 Node 데몬 등록 + 온보딩(로그인 항목 안내). |
| 정밀도 평가 리포트 | 소량 골드셋(30~200개, 클래스 각 30%+) 기반 accuracy/precision/recall/F1 + Cohen's κ + Balanced Accuracy 자동 기록. |
| 녹화 리플레이 (데모 백업) | 기록된 JSONL을 타임스탬프 순으로 재주입 + LLM 응답 모킹으로 결정론적 재현. 라이브 시연이 평가자 환경에서 흔들릴 때를 대비한 안전장치. |
| **(명시적 비범위)** 자동 차단/개입 | PreToolUse deny·Stop block을 통한 자동 차단/일시중지는 **후속 과제**. MVP는 탐지+알림+사람호출까지만. |

### 6-2 차별화 요소

**기존 서비스 문제점:**
시장은 대부분 **관측·평가(observe & evaluate)** 에 머문다. 트레이스를 보여주거나(관측), 사후에 품질을 채점하거나(평가), 토큰 한도를 거는(게이트웨이) 식이다. "코딩 에이전트가 *지금* 행동상 헛돌고 있다"를 실시간으로 의미 판정해 **근거와 함께 사람을 부르는** 조합은 아직 상대적으로 빈자리다. (절대 공백은 아니다 — AgentOps가 가장 가까운 방향이나 루프 탐지는 미출시 로드맵 단계.)

**내가 개선할 부분 (3가지 교집합):**
1. **코딩 에이전트 특화 실패패턴의 의미적 분류** — 헛돌기·가짜 성공. 보안 가드레일(Straiker)·일반 품질 평가(Galileo 등)와 측정 대상이 다르다.
2. **Claude Code 로그/훅 네이티브 실시간 통합** — JSONL 실시간 테일링. (단 이건 공개 API라 진입장벽이 낮으므로 "해자"로 주장하지 않는다. 해자는 탐지 품질과 에스컬레이션 UX에 둔다.)
3. **단순 차단이 아닌 "근거 동반 사람 호출"** — 무엇이·왜 의심스러운지를 근거와 함께 전달해 사람이 판단하게 한다.

> **정직한 한계:** 차별화의 핵심은 "실시간 개입 자체"가 아니라 위 3가지 교집합이다. 훅 접근성은 누구나 쓸 수 있으므로 경쟁사·인접 제품(특히 AgentOps)이 좁혀올 수 있다. 방어선은 "탐지 정확도와 에스컬레이션 경험"이다.

## 7. 사용할 AI 기술 조사

### 7-1 사용할 알고리즘

| 알고리즘 | 사용 이유 |
|---|---|
| **(tool, args) 시퀀스 반복 탐지** (슬라이딩 윈도 + SHA-256 해시) | 1차 저비용 게이트. "문자 그대로의 반복"을 LLM 호출 없이 빠르게 거른다. 기성 도구 임계값 참고(예: warning 10 / critical 20). |
| **임베딩 코사인 유사도 + 밀도기반 군집화 (HDBSCAN / SemDeDup식 k-means+코사인)** | 입력이 미세 변형돼 규칙을 빠져나가는 의미적 헛돌기·silent cycle을 잡는다. 클러스터 수 사전 지정 불필요. 임계값은 자체 골드셋으로 캘리브레이션(실무 0.85~0.95). |
| **LLM-as-judge (루브릭 + CoT + self-consistency)** | "근거 없는 완료 선언/자기승인 우회" 같은 가짜 성공은 의미를 읽어야만 판정 가능. few-shot 예시·다차원 루브릭으로 정확도 향상. **편향 방어 필수:** position/verbosity/self-preference 편향이 체계적으로 존재(arXiv:2406.07791) → position swapping·permutation·다수결로 완화. |
| **소량 골드셋 분류 평가 (precision/recall/F1 + Cohen's κ + Balanced Accuracy)** | judge 정렬을 분류 문제로 보고 정량 측정. 단일 accuracy는 실패 클래스를 놓치므로 금지. 클래스 불균형 보정을 위해 Balanced Accuracy 병행. κ는 ≥0.80 strong / 0.60~0.79 substantial 해석. |

**근거가 되는 정량 연구 (방법론·메트릭 차용):**
- **IBM Unsupervised Cycle Detection** (arXiv:2511.10650, ICPE 2026): 구조(1차) + 의미 유사도(2차) **하이브리드가 F1 0.72**(P 0.62 / R 0.86), 구조만 0.08·의미만 0.28. → LoopBreaker의 2단계 아키텍처를 직접 뒷받침.
- AgentProcessBench / TRAJECT-Bench: human-labeled step 데이터로 "실패 트래젝토리는 오류 스텝 비율이 일관되게 높다"를 입증 → 라벨링·메트릭 방법론 차용.

> **정직한 한계 (기획서에 명시):** 가장 강한 외부 근거인 IBM 하이브리드도 **F1 0.72 = 오탐 38%**, 단일 도메인·진행 중(WIP) 연구다. 따라서 정량 목표를 박기보다 "오탐율을 관리 가능한 수준으로 유지"를 정성 목표로 삼고, 골드셋 측정으로 실제 수치를 보고한다.

## 8. 기술 스택 조사

### 8-1 개발 언어

- **TypeScript / Node.js** (메인 런타임 — 사용자 숙련도 일치, 단일화)
- (보조) Python — 로컬 임베딩을 쓸 경우에만. **본 MVP는 임베딩·judge를 API로 외주**하므로 Node 단일화 유지.

### 8-2 사용 라이브러리

| 라이브러리 | 사용 목적 |
|---|---|
| `chokidar` | 파일/디렉터리 실시간 감시(재귀, macOS 이벤트 정규화). `fs.watch`는 macOS 파일명 미보고·중복으로 부적합. |
| Node 내장 `readline` (+ 필요 시 `stream-json`/`ndjson`) | JSONL 증분 파싱. 바이트 오프셋 추적으로 성장 파일 tail. |
| `@anthropic-ai/sdk` (또는 OpenAI SDK) | LLM-as-judge 호출(`messages.stream()`), 임베딩 API. 재시도·타임아웃·취소 내장. **prompt caching**으로 비용 절감(캐시 읽기 ≈ 입력 단가의 10%). |
| `better-sqlite3` + `sqlite-vec` | 로컬 단일 SQLite 파일에 이벤트·임베딩·판정 저장 + 벡터 검색(별도 벡터DB 불필요). |
| `node-notifier` (또는 유지보수 fork / Electron Notification) | macOS Notification Center 데스크톱 알림. |
| `launchd` LaunchAgent (시스템) | per-user 백그라운드 데몬 등록(로그인 시 실행, GUI 알림 접근). |

### 8-3 개발 환경

- **VSCode** (메인 에디터)
- **GitHub** (버전 관리·이슈)
- **macOS** (개발·실행 타깃 — MVP는 macOS 전용. Windows/Linux는 향후 확장)
- **Node.js 20+** / npm
- (데모) 기록 JSONL 리플레이 하니스 — 라이브 시연 + 결정론적 재현 백업

> **개인정보·프라이버시 고지:** 세션 로그에는 코드·프롬프트가 포함된다. LLM-judge·임베딩을 외부 API로 보내므로, MVP에서는 **(1) 무엇이 전송되는지 명시 + (2) 전송 전 민감정보 필터/옵트인** 을 둔다. 데이터 본체는 로컬 SQLite에 머문다.
